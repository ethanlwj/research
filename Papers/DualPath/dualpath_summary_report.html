DualPath: Breaking the Storage Bandwidth Bottleneck in Agentic LLM Inference

Easy-to-read technical summary with key ideas, challenges, results, and practical lessons.

Source: arXiv:2602.21548 · HTML version

1) Abstract (Plain-English)

The paper studies a systems bottleneck in agentic LLM serving: repeated KV-cache movement from remote storage can dominate end-to-end performance. DualPath introduces a dual-path loading and scheduling design that uses both prefill and decode sides more effectively, so storage I/O is no longer concentrated on prefill engines only. The reported result is a substantial gain in throughput and latency under realistic multi-turn agent workloads.

2) Introduction: Why This Problem Exists

  • Agentic workloads are multi-turn and tool-driven, often with many short appends over long contexts.
  • This creates high KV-cache reuse, so loading KV-cache efficiently becomes more important than raw compute speed.
  • In existing prefill-decode disaggregated systems, prefill-side storage bandwidth becomes the limiting resource while decode-side bandwidth is often underused.

3) Main Contributions

Storage bottleneck reframing

Shows storage-network pressure is the primary bottleneck in agentic settings, not just GPU compute.

Dual-path loading design

Adds a second data path so KV-cache movement can use spare bandwidth on decode-side resources.

NIC-centric traffic isolation

Separates KV-cache traffic from latency-sensitive model communications to reduce interference.

Dynamic scheduling

Balances both NIC traffic and GPU utilization across engines for stable high throughput.

4) Key Challenges

  • How to exploit spare decode-side I/O without harming core inference communications (AllToAll, AllGather/ReduceScatter).
  • How to avoid introducing new congestion on compute links and PCIe paths in the dual data path.
  • How to maintain load balance simultaneously across storage NICs and GPUs at scale.

5) Problems Solved

  • Breaks prefill-side storage bottleneck by aggregating usable bandwidth across prefill and decode paths.
  • Reduces idle capacity on decode-side storage resources.
  • Improves system-level efficiency for long-context, multi-agent workloads where cache movement dominates.

6) Results (As Reported in the Paper)

Up to 1.73× throughput Up to 1.61× lower median latency Up to 90.9% less storage bandwidth usage ~77.3% speculative token acceptance rate

Evaluation includes offline and online settings, and scales up to large GPU counts. The paper positions these gains as evidence that storage-aware serving architecture is a first-class optimization target for agentic inference.

8) Summary

DualPath is a systems-level solution for agentic LLM serving where KV-cache I/O, not compute, dominates. Its core value is practical: it turns underused decode-side storage capacity into useful throughput, while scheduling and traffic isolation keep inference stable.

9) Actionable Lessons

  • Measure storage-network saturation and KV-cache transfer cost before only scaling GPUs.
  • Treat prefill and decode as a joint bandwidth-management problem, not independent queues.
  • Isolate cache traffic from latency-critical collective ops using explicit NIC/traffic policy.
  • Use agent-trace benchmarks (long context + many turns) to evaluate serving architecture decisions.
If your serving stack shows high cache hit rate and prefill-side network saturation, DualPath-style data-path and scheduling changes likely deliver higher ROI than pure compute-side tuning.
